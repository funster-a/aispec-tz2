## 🚀 Как это работает: Архитектура

Проект построен по принципу монорепозитория, включающего 3 независимых компонента:

1. **`dashboard/`** — Фронтенд на базе Next.js (App Router), Tailwind CSS v4 и Recharts. Реализован "премиальный" тёмный дизайн. 
2. **`scripts/`** — Node.js скрипты:
   - `upload-to-retailcrm.js`: парсит `mock_orders.json` и отправляет заказы в RetailCRM через POST-запрос.
   - `sync-to-supabase.js`: забирает заказы из RetailCRM (постранично) и делает `upsert` в Supabase БД напрямую через API.
3. **`bot/telegram-bot.js`** — Бот-пуллер, который сверяет заказы в RetailCRM за последние 60 секунд и, если есть чеки выше `50 000 ₸`, шлёт уведомление.

## 🤖 Работа с AI: Промпты, Затыки и Решения

Вся разработка этого проекта велась через продвинутый AI ассистент (Claude/Gemini). Ниже описана статистика взаимодействия:

### Что было передано в ИИ:
- Структура исходно желаемого проекта (монорепа, Tailwind, Next.js).
- Специфические требования к дашборду (3 карточки KPI, двойной график Bar+Line, Recharts).
- Требования к Supabase SQL таблицам и индексам.
- Требования к Node.js Polling боту в ТГ.

### С какими проблемами мы столкнулись (Затыки) и как AI их решил:

1. **Затык 1: Ошибка валидации RetailCRM при загрузке данных (`Input value "order" contains a non-scalar value`)**
   - *Проблема:* RetailCRM API требует, чтобы объект заказа передавался в `x-www-form-urlencoded` как строка (строгий JSON), а первоначальный скрипт пытался передавать вложенные поля (например `order[delivery][city]`). Из-за этого CRM отдавал HTTP 400.
   - *Решение:* AI оперативно переписал сборку параметров: весь payload заказа был свернут в один `order` параметр через `JSON.stringify()`. Загрузка всех 50 демо-заказов прошла успешно.

2. **Затык 2: Ошибки 404 при синхронизации CRM → Supabase**
   - *Проблема:* Из переменной `RETAILCRM_API_URL` приходил URL со слэшем на конце (`https://.../`), а AI скрипт добавлял свой слэш, получался двойной слэш `//api/v5/orders`, на что сервер CRM отдавал `404 Not Found`.
   - *Решение:* Написан фильтр через `replace(/\/$/, '')` для базы URL, чтобы ссылки формировались корректно.

3. **Затык 3: Падение Telegram-бота (`ETELEGRAM: 400 Bad Request... Character '.' is reserved`)**
   - *Проблема:* Бот успешно находил крупные чеки, но при отправке ломался. Парсер `MarkdownV2`, который использовал бот по умолчанию, был слишком чувствителен к точкам, запятым и тире из заказа.
   - *Решение:* AI сменил `parse_mode` с `MarkdownV2` на `HTML`, полностью решив конфликты с экранированием символов (использованы `<b>`, `<code>`).

4. **Затык 4: Ошибка сборки Next.js и конфликт Tailwind CSS v4**
   - *Проблема:* Фронтенд не мог сбилдиться из-за новой системы Tailwind. К тому же во время деплоя/билда срабатывал API-роут для Supabase до того, как переменные окружения были готовы.
   - *Решение:* Установлен правильный `@tailwindcss/postcss`, CSS-директивы заменены на новые `@import "tailwindcss"`, а вызов Supabase клиента в API вынесен внутрь функции обработчика (Lazy init).

AI-менеджмент позволил в кратчайшие сроки (около 10-15 минут чистого времени) полностью создать архитектуру, найти неочевидные проблемы в специфических API (RetailCRM) и адаптировать код под релиз.
